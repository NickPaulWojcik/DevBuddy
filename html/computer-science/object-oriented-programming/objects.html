What are objects?

Objects are the core of any object oriented programming language. They provide developers building blocks to create complex and structured code. Objects in programming are designed after objects that can be found in the real world.

Objects are broken up into two major uses: variables, and functions.

  • variables: data stored within the object
  • functions: actions the object can take

Objects are created by instantiation. When an object is instantiated, it makes an independent copy. At anytime there may be multiple independent objects of the same type.

Each object contains its own variables and functions. For example, we can have a lamp object with an integer variable that tracks if its on or off with the numbers 0 for off and 1 for on. This object can also have two functions, one for turning the lamp on and one for turning it off. If a developer creates two lamp objects, say A and B, the developer can call the turn on function of lamp A and the turn off function of lamp B. If the developer checks the variable used to track if its on or off, they will see that lamp A is on and lamp B is off.

How are objects used?

Since objects are built to match real world objects, they can be used to design complex software in a familiar way. Most software aims to solve problems in the real world, which consists of real objects, which means developers can design their systems with recognizable objects.

Its important to keep in mind that not all objects developers come across will be based on an object in the real world, but most will. For example, a developer may want to make an object that just has functions to solve math problems and then use that object repeatidly without writing new code each time.
